## 🚨 The Core Concept: Cross-Origin Resource Sharing (CORS)

When your browser sees a request coming from `localhost:5500` (the frontend) and trying to talk to `localhost:8000` (the backend), it immediately blocks it by default due to security rules 
(the Same-Origin Policy).

The only way to unblock this is to modify the **backend** to include special HTTP response headers (like `Access-Control-Allow-Origin`) that explicitly grant permission to the frontend's 
origin.

## ✨ Advanced Alternative: Using a Proxy (The Cleanest Fix)

If you are using a modern frontend build tool (like **Vite**, **Create React App**, or **Webpack Dev Server**), the **best practice** is often *not* to change the backend CORS settings but 
rather to tell the **frontend development server** to simply redirect the requests.

This makes the request appear to the browser as if it originated from the same domain/port, bypassing the CORS issue entirely during development.

### Example: Vite Proxy Setup
If your frontend is built with Vite, you can modify your `vite.config.js`:

```javascript
// vite.config.js
import { defineConfig } from 'vite'
import react from '@vitejs/plugin-react'

export default defineConfig({
  plugins: [react()],
  server: {
    proxy: {
      // Any request starting with /api will be redirected to the backend
      '/api': {
        target: 'http://localhost:8000', 
        changeOrigin: true, // Important for proper CORS headers
        rewrite: (path) => path.replace(/^\/api/, ''),
      },
    },
  },
})
```
**Result:** When the frontend code calls `/api/data`, Vite intercepts it and routes it directly to `http://localhost:8000/api/data` *before* the browser ever sees the cross-origin call.
